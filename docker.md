# Docker Commands Reference Guide

A comprehensive cheat-sheet of frequently used Docker commands, categorized by function, with realistic examples and detailed explanations.

---

## 1. Container Lifecycle Management

These commands control the creation, execution, state, and destruction of Docker containers.

### `docker run`
Creates and starts a new container from an image.
* **Syntax:** `docker run [OPTIONS] IMAGE [COMMAND] [ARG...]`
* **Example:** Run a detached Nginx web server mapping port 8080 on the host to port 80 inside the container, named "my-web-server":
    ```bash
    docker run -d -p 8080:80 --name my-web-server nginx:latest
    ```
* **Interactive Example:** Run an interactive Ubuntu container and open a bash shell (automatically removed after exiting):
    ```bash
    docker run -it --rm ubuntu:22.04 /bin/bash
    ```

### `docker start`
Starts one or more stopped containers.
* **Syntax:** `docker start [OPTIONS] CONTAINER [CONTAINER...]`
* **Example:** Restart a previously stopped container named "my-web-server":
    ```bash
    docker start my-web-server
    ```

### `docker stop`
Stops one or more running containers gracefully by sending `SIGTERM`, then `SIGKILL` after a grace period.
* **Syntax:** `docker stop [OPTIONS] CONTAINER [CONTAINER...]`
* **Example:** Stop the running container "my-web-server":
    ```bash
    docker stop my-web-server
    ```

### `docker restart`
Stops and then starts a container again.
* **Syntax:** `docker restart [OPTIONS] CONTAINER [CONTAINER...]`
* **Example:** Apply configuration updates by restarting the database container:
    ```bash
    docker restart my-postgres-db
    ```

### `docker pause` / `docker unpause`
Suspends all processes within one or more containers.
* **Syntax:** `docker pause CONTAINER` / `docker unpause CONTAINER`
* **Example:** Freeze container execution temporarily:
    ```bash
    docker pause my-web-server
    docker unpause my-web-server
    ```

### `docker rm`
Removes one or more stopped containers.
* **Syntax:** `docker rm [OPTIONS] CONTAINER [CONTAINER...]`
* **Example:** Delete a stopped container:
    ```bash
    docker rm my-web-server
    ```
* **Force-Delete Running Example:** Forcefully remove a running container (`-f` sends `SIGKILL`):
    ```bash
    docker rm -f my-web-server
    ```

## 2. Container Inspection & Interacting

Use these commands to view logs, execute commands inside containers, and monitor performance metrics.

### `docker ps`
Lists containers. By default, it shows only running containers.
* **Syntax:** `docker ps [OPTIONS]`
* **Example:** List all active running containers:
    ```bash
    docker ps
    ```
* **All Containers Example:** List all containers regardless of state (running, exited, created):
    ```bash
    docker ps -a
    ```

### `docker logs`
Fetch logs generated by a container.
* **Syntax:** `docker logs [OPTIONS] CONTAINER`
* **Example:** View the latest 50 lines of logs and follow output updates in real-time (`-f`):
    ```bash
    docker logs -f --tail 50 my-web-server
    ```

### `docker exec`
Runs a new command inside an already running container.
* **Syntax:** `docker exec [OPTIONS] CONTAINER COMMAND [ARG...]`
* **Example:** Open an interactive bash shell terminal inside a running Alpine container:
    ```bash
    docker exec -it my-alpine-container /bin/sh
    ```
* **Non-interactive execution:** Run a specific command without entering the container:
    ```bash
    docker exec my-postgres-db psql -U postgres -c "SELECT 1;"
    ```

### `docker inspect`
Returns low-level, detailed JSON information on Docker objects (containers, images, volumes, networks).
* **Syntax:** `docker inspect [OPTIONS] NAME|ID [NAME|ID...]`
* **Example:** Extract the specific IP address of a running container:
    ```bash
    docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' my-web-server
    ```

### `docker top`
Displays the running processes inside a specific container.
* **Syntax:** `docker top CONTAINER [ps OPTIONS]`
* **Example:** Check active PID lists for a container:
    ```bash
    docker top my-web-server
    ```

### `docker stats`
Displays a live, streaming metric dashboard of container(s) resource usage (CPU, Memory, Network I/O).
* **Syntax:** `docker stats [OPTIONS] [CONTAINER...]`
* **Example:** Monitor resource consumption of all running containers:
    ```bash
    docker stats
    ```

## 3. Image Management

Commands used to pull, build, view, tag, and remove Docker images locally.

### `docker build`
Builds an image from a standard `Dockerfile`.
* **Syntax:** `docker build [OPTIONS] PATH | URL | -`
* **Example:** Build an image tagged as `myapp:v1.0` using the `Dockerfile` located in the current workspace path (`.`):
    ```bash
    docker build -t myapp:v1.0 .
    ```

### `docker images` / `docker image ls`
Lists all locally stored Docker images.
* **Syntax:** `docker images [OPTIONS] [REPOSITORY[:TAG]]`
* **Example:** Check existing local images:
    ```bash
    docker images
    ```

### `docker pull`
Downloads an image from a remote registry (like Docker Hub) to your local machine.
* **Syntax:** `docker pull [OPTIONS] NAME[:TAG|@DIGEST]`
* **Example:** Download the latest official Redis image:
    ```bash
    docker pull redis:latest
    ```

### `docker push`
Uploads a local image or repository to a remote registry.
* **Syntax:** `docker push [OPTIONS] NAME[:TAG]`
* **Example:** Push a tagged image to your personal Docker Hub account registry:
    ```bash
    docker push username/myapp:v1.0
    ```

### `docker tag`
Creates a target alias tag that refers back to a source image.
* **Syntax:** `docker tag SOURCE_IMAGE[:TAG] TARGET_IMAGE[:TAG]`
* **Example:** Tag an existing image for an internal repository or registry path:
    ```bash
    docker tag local-app:latest myregistry.company.com/production/app:v1.0
    ```

### `docker rmi`
Removes one or more images from the local repository cache.
* **Syntax:** `docker rmi [OPTIONS] IMAGE [IMAGE...]`
* **Example:** Delete an unused node image:
    ```bash
    docker rmi node:18-alpine
    ```

## 4. Volume & Network Configuration

These primitives allow data persistence and isolated environment routing setup.

### `docker volume create`
Creates a persistent storage volume managed entirely by Docker engine.
* **Syntax:** `docker volume create [OPTIONS] [VOLUME]`
* **Example:** Provision a storage space named `db-data`:
    ```bash
    docker volume create db-data
    ```
* **Usage with `docker run`:** Mount that volume inside a container:
    ```bash
    docker run -d -v db-data:/var/lib/postgresql/data postgres:15
    ```

### `docker volume ls`
Lists all volumes currently recognized by the Docker daemon.
* **Syntax:** `docker volume ls [OPTIONS]`
* **Example:** Locate orphaned or managed local storage volumes:
    ```bash
    docker volume ls
    ```

### `docker network create`
Creates a logical network overlay/bridge so containers can communicate directly with each other securely.
* **Syntax:** `docker network create [OPTIONS] NETWORK`
* **Example:** Spin up a new bridge-isolated environment:
    ```bash
    docker network create backend-network
    ```
* **Usage with `docker run`:** Assign container to the network:
    ```bash
    docker run -d --network backend-network --name api-service my-api:latest
    ```

### `docker network ls`
Lists all logical overlay/bridge configurations configured inside Docker.
* **Syntax:** `docker network ls [OPTIONS]`
* **Example:** Verify customized or default routing endpoints:
    ```bash
    docker network ls
    ```

## 5. System Cleanups & Maintenance

Crucial commands to free up storage space and remove dangling configurations.

### `docker system prune`
Removes all unused data across containers, networks, images (dangling layers), and optionally volumes.
* **Syntax:** `docker system prune [OPTIONS]`
* **Example:** Clear standard cache safely (asks for confirmation):
    ```bash
    docker system prune
    ```
* **Deep Purge Example:** Force fully wipe out all unused images, stopped containers, and volumes altogether:
    ```bash
    docker system prune -a --volumes -f
